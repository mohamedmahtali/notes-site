# .gitlab-ci.yml

## Parent
- [[GitLab CI]]

---

## Définition

`.gitlab-ci.yml` est le fichier de configuration GitLab CI/CD placé à la racine du repo. Il définit les stages, les jobs, et les règles qui composent le pipeline.

---

## Structure complète

```yaml
# Étapes globales (ordre d'exécution)
stages:
  - build
  - test
  - security
  - deploy

# Variables globales
variables:
  DOCKER_DRIVER: overlay2
  IMAGE: $CI_REGISTRY_IMAGE:$CI_COMMIT_SHA

# Template réutilisable
.base-job: &base
  image: node:20-alpine
  cache:
    paths:
      - node_modules/

# Héritage du template
test:unit:
  <<: *base
  stage: test
  script:
    - npm ci
    - npm run test:unit
  artifacts:
    reports:
      coverage_report:
        coverage_format: cobertura
        path: coverage/cobertura-coverage.xml

# Job conditionnel
deploy:production:
  stage: deploy
  script:
    - ./deploy.sh
  environment:
    name: production
  rules:
    - if: $CI_COMMIT_BRANCH == "main"
      when: manual
    - when: never
```

---

## Inclusions

```yaml
# Inclure d'autres fichiers de config
include:
  - local: '.gitlab/ci/build.yml'
  - project: 'devops/ci-templates'
    ref: main
    file: '/templates/docker.yml'
```

---

> [!tip]
> Utiliser `include` pour partager des templates CI entre plusieurs projets GitLab. Centralise la logique CI dans un repo dédié.
